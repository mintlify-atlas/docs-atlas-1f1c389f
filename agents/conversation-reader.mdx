---
title: "Conversation Reader"
description: "Read full iMessage and email history for any contact"
---

The Conversation Reader agent retrieves and displays the complete message history for a specific contact across both iMessage and Apple Mail, with rich text extraction, attachment references, and tapback reactions.

## When to Use

This agent is triggered when you provide a contact name, phone number, or email address and want to read messages:

- "Show me my texts with Sarah"
- "Messages from +1234567890"
- "Emails from Jane"
- "All communications with John"

**Usage patterns:**

```
/imessage-mail texts with Sarah
/imessage-mail messages from +1234567890
/imessage-mail all communications with John
```

## Workflow

<Steps>

### Resolve the Contact

**If the user provides a name**, the agent first looks up matching phone numbers from the Address Book:

```sql
SELECT T2.ZFULLNUMBER, T1.ZFIRSTNAME, T1.ZLASTNAME
FROM ZABCDRECORD AS T1
INNER JOIN ZABCDPHONENUMBER AS T2 ON T1.Z_PK = T2.ZOWNER
WHERE LOWER(T1.ZFIRSTNAME) LIKE LOWER('%<name>%')
   OR LOWER(T1.ZLASTNAME) LIKE LOWER('%<name>%')
   OR LOWER(T1.ZFIRSTNAME || ' ' || T1.ZLASTNAME) LIKE LOWER('%<name>%');
```

And their email address for Mail lookups:

```sql
SELECT T2.ZADDRESS, T1.ZFIRSTNAME, T1.ZLASTNAME
FROM ZABCDRECORD AS T1
INNER JOIN ZABCDEMAILADDRESS AS T2 ON T1.Z_PK = T2.ZOWNER
WHERE LOWER(T1.ZFIRSTNAME) LIKE LOWER('%<name>%')
   OR LOWER(T1.ZLASTNAME) LIKE LOWER('%<name>%')
   OR LOWER(T1.ZFIRSTNAME || ' ' || T1.ZLASTNAME) LIKE LOWER('%<name>%');
```

The resolved number is then verified in `chat.db`:

```bash
sqlite3 ~/Library/Messages/chat.db "SELECT id FROM handle WHERE id LIKE '%<last_7_digits>%';"
```

<Info>
If multiple contacts match the name, the agent presents the choices and asks you to select one. If a phone number or email is provided directly, it is used as-is.
</Info>

### Fetch iMessages

The agent queries the last 25 messages (configurable) from `chat.db`, including any attachment filenames:

```sql
SELECT
  m.ROWID,
  CASE WHEN m.is_from_me = 1 THEN 'Me' ELSE 'Them' END as sender,
  COALESCE(m.text, CASE WHEN m.attributedBody IS NOT NULL THEN '[Rich Text/Hidden]' ELSE '[No Text]' END) as content,
  datetime(m.date / 1000000000 + 978307200, 'unixepoch', 'localtime') as timestamp,
  a.filename,
  a.mime_type
FROM message m
JOIN handle h ON m.handle_id = h.ROWID
LEFT JOIN message_attachment_join maj ON m.ROWID = maj.message_id
LEFT JOIN attachment a ON maj.attachment_id = a.ROWID
WHERE h.id LIKE '%<digits_only>'
ORDER BY m.date DESC
LIMIT <count>;
```

### Fetch Emails from Apple Mail (parallel)

If the contact has an email address, a parallel subagent fetches messages received **from** them:

```sql
SELECT
  m.ROWID,
  CASE WHEN mb.url LIKE '%Sent%' THEN 'Me' ELSE a.comment END as sender,
  sub.subject,
  datetime(m.date_received, 'unixepoch', 'localtime') as received,
  CASE WHEN m.read = 1 THEN 'Read' ELSE 'UNREAD' END as status,
  m.size
FROM messages m
LEFT JOIN addresses a ON m.sender = a.ROWID
LEFT JOIN subjects sub ON m.subject = sub.ROWID
LEFT JOIN mailboxes mb ON m.mailbox = mb.ROWID
WHERE (a.address LIKE '%<email_or_domain>%' OR a.comment LIKE '%<contact_name>%')
  AND m.deleted = 0
ORDER BY m.date_received DESC
LIMIT 20;
```

And messages **sent to** them:

```sql
SELECT
  m.ROWID,
  'Me' as sender,
  sub.subject,
  datetime(m.date_sent, 'unixepoch', 'localtime') as sent,
  m.size
FROM messages m
JOIN recipients r ON r.message = m.ROWID
JOIN addresses a ON r.address = a.ROWID
LEFT JOIN subjects sub ON m.subject = sub.ROWID
WHERE (a.address LIKE '%<email>%' OR a.comment LIKE '%<contact_name>%')
  AND m.deleted = 0
ORDER BY m.date_sent DESC
LIMIT 20;
```

### Read Email Body Content

The Envelope Index database does not store email body text. To read a full email, the agent locates its `.emlx` file on disk:

```bash
find ~/Library/Mail/V10 -name "<MESSAGE_ROWID>.emlx" -o -name "<MESSAGE_ROWID>.partial.emlx" 2>/dev/null | head -1
```

Then reads it (skipping the byte-count header on the first line):

```bash
cat "$(find ~/Library/Mail/V10 -name '<MESSAGE_ROWID>.emlx' 2>/dev/null | head -1)" | tail -n +2
```

For HTML-heavy emails, the agent strips tags to extract readable plain text:

```bash
cat "$(find ~/Library/Mail/V10 -name '<MESSAGE_ROWID>.emlx' 2>/dev/null | head -1)" | tail -n +2 | sed -n '/<body/,/<\/body>/p' | sed 's/<[^>]*>//g' | head -50
```

### Get Email Attachments

```sql
SELECT att.name
FROM attachments att
WHERE att.message = <MESSAGE_ROWID>;
```

### Extract Rich Text Bodies (iMessage)

When an iMessage shows `[Rich Text/Hidden]`, the text is stored as a binary `NSAttributedString` blob in the `attributedBody` column rather than in `text`. The agent extracts it with:

```bash
for ROWID in <ROWID1> <ROWID2> <ROWID3>; do
  echo "---ROWID:$ROWID---"
  sqlite3 ~/Library/Messages/chat.db "SELECT HEX(attributedBody) FROM message WHERE ROWID = $ROWID;" | xxd -r -p | strings | head -3
done
```

This is run in a single batch subagent for all rich-text messages at once, then the `[Rich Text/Hidden]` placeholders in the output are replaced with the real text.

<Note>
See [Architecture](/reference/architecture) for more detail on the `attributedBody` format and the Mac CoreData timestamp epoch.
</Note>

### Handle Tapback Reactions (optional)

iMessage tapback reactions are stored as separate message rows with an `associated_message_type`. The agent can query them:

```sql
SELECT
  m.ROWID,
  m.associated_message_type,
  m.associated_message_guid
FROM message m
JOIN handle h ON m.handle_id = h.ROWID
WHERE h.id LIKE '%<digits_only>'
  AND m.associated_message_type != 0
ORDER BY m.date DESC
LIMIT 20;
```

Tapback type codes:

| Code | Reaction |
|------|----------|
| 2000 | Love |
| 2001 | Like |
| 2002 | Dislike |
| 2003 | Laugh |
| 2004 | Emphasis |
| 2005 | Question |

</Steps>

## Output Format

iMessages and emails are shown in separate sections by default, or interleaved chronologically if you ask for "all communications":

```
## Conversation with John Smith

### iMessages (+1234567890)
*25 most recent*

**14:30 Me**: Hey, are you free tonight?
**14:32 John**: Yeah! What's up?
**14:33 Me**: Check out this photo 📎 IMG_1234.jpg
**14:35 John**: 😂 That's hilarious

---

### Emails (john@example.com)
*10 most recent*

| Status | Direction | Subject | Date |
|--------|-----------|---------|------|
| Read | From John | Re: Project Update | Mar 16, 10:00 |
| Read | To John | Project Update | Mar 15, 14:30 |
| UNREAD | From John | Invoice #1234 | Mar 14, 09:00 |
```

iMessages use the conversational format by default. Request a table if you prefer a more compact view.
