This guide walks you through installing the skill, granting the required macOS permissions, and running your first command.

## Prerequisites

Before you start, make sure you have:

- **Claude Code** CLI installed
- **macOS 15+** (tested on Apple Silicon)
- **Messages.app** signed into iMessage
- **Mail.app** with at least one account configured

<Note>
  Full Disk Access is required for the skill to read your local message databases. You will grant this in step 2.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the skill repository to your local machine:

    ```bash
    git clone https://github.com/stat-guy/imessage-mail.git
    ```
  </Step>

  <Step title="Copy the skill files">
    Create the skills directory and copy the files into it:

    ```bash
    mkdir -p ~/.claude/skills/imessage-mail
    cp -r imessage-mail/SKILL.md ~/.claude/skills/imessage-mail/
    cp -r imessage-mail/agents ~/.claude/skills/imessage-mail/
    ```

    Your skills directory should look like this:

    ```
    ~/.claude/skills/imessage-mail/
    ├── SKILL.md
    └── agents/
        ├── recent-activity.md
        ├── conversation-reader.md
        ├── message-search.md
        ├── attachment-finder.md
        ├── mail-reader.md
        └── send-message.md
    ```
  </Step>

  <Step title="Grant Full Disk Access to your terminal">
    The skill reads SQLite databases that macOS protects with Full Disk Access. You must add your terminal application before the skill can query them.

    1. Open **System Settings** > **Privacy & Security** > **Full Disk Access**
    2. Click the **+** button
    3. Add your terminal app (Terminal.app, iTerm2, Warp, Ghostty, etc.)
    4. Restart the terminal

    <Warning>
      Without Full Disk Access, all database queries will fail with a permission error. This step is required — there is no workaround.
    </Warning>
  </Step>

  <Step title="Allow AppleScript automation (send only)">
    The first time you ask the skill to send an iMessage or email, macOS will show a one-time dialog asking whether to allow your terminal to control **Messages.app** or **Mail.app**. Click **Allow**.

    <Note>
      If you accidentally click Deny, you can re-enable it in **System Settings** > **Privacy & Security** > **Automation**.
    </Note>
  </Step>
</Steps>

## Run your first command

With the skill installed and Full Disk Access granted, open Claude Code and run:

```
/imessage-mail recent
```

Claude will query your Messages and Mail databases and return an overview of recent activity:

## What to try next

Here are a few more commands to explore what the skill can do:

```
/imessage-mail messages from Sarah
/imessage-mail search "project deadline"
/imessage-mail email
/imessage-mail send a text to Mike saying I'm running late
/imessage-mail find attachments from David
```

<Tip>
  The skill understands natural language — you don't need to memorize exact command syntax. Try phrasing requests the way you'd say them out loud.
</Tip>
